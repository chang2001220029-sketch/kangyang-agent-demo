# 乐颐 · 康养 Agent 演示说明与体验路径

> 面向领导汇报的现场演示脚本。本系统在 `音视频识别.html` 基础上改造，
> 以**写实 3D 虚拟数字人「乐颐」**为交互入口（真人级面部、说话口型实时同步、随情绪变表情），
> 现场演示康养场景 7 个智能体中最具代表性的 3 个能力。

---

## 一、系统一句话介绍

「乐颐」是一个带虚拟数字人形象的康养智能体演示系统：老人对着屏幕说话，数字人能**看懂表情、听懂话、识别药品、发现心理危机**，并把每一步判断以**结构化风险输出**（风险等级 / 置信度 / 证据 / 建议动作 / 是否需人工复核）实时展示在右侧「Agent 决策面板」，直观呈现我们智能体的技术能力与安全边界。

---

## 二、7 个 Agent 能力全景（对应 PRD）

| 序号 | Agent | 核心职责 | 本次演示是否直接展示 |
| --- | --- | --- | --- |
| 1 | 健康预警 AI 小模型 | 多源健康数据异常检测与风险分级 | 场景 C 中作为服药前后指标关联体现 |
| 2 | 药事管理 Agent | 用药计划、提醒、服药识别、错服/漏服阻断 | ✅ 场景 C 主角 |
| 3 | 情绪识别模型（多模态） | 语音/文字/表情/行为融合识别情绪 | ✅ 场景 A / B 主角 |
| 4 | 心理危机干预 Agent | 危机识别、分级、安抚话术、人工转介 | ✅ 场景 B 主角 |
| 5 | 认知能力评估模型 | 语音/文字认知初筛与趋势 | 能力条展示（本次不深入演示） |
| 6 | 服务调度推荐 Agent | 按事件与资源生成派单/推荐 | ✅ 场景 B 中危机联动派单 |
| 7 | 情绪陪伴 AI 助手 | 语音陪伴、内容推荐、生活提醒、风险识别 | ✅ 场景 A 主角 |

> 顶部「7-Agent 能力条」会随场景高亮当前激活的 Agent，让领导一眼看到"这一步是哪个智能体在工作"。

统一风险等级（PRD 统一口径）：
- **P3 常规**（绿）：日常，正常记录
- **P2 关注**（黄）：轻度异常，提醒关注
- **P1 高风险**（橙）：异常明确，通知人工确认
- **P0 紧急**（红）：急迫安全风险，立即触发处理流程

## 三、演示前准备（5 分钟）

1. **打开方式**：为使用摄像头/麦克风，需通过 `localhost` 或 HTTPS 打开。
   - 在项目根目录（`语音视频对话/`）运行本地服务器：
     ```bash
     python3 serve.py
     ```
   - 浏览器打开：`https://localhost:8443/康养Agent演示/index.html`
     （自签名证书会提示不安全，点「高级 → 继续访问」）
   - 或本机直接：`http://localhost:8000/康养Agent演示/index.html`（用 `python3 -m http.server 8000`）
2. **浏览器**：Chrome 或 Edge（语音识别依赖 Web Speech API；3D 数字人依赖 WebGL，现代浏览器均支持）。
3. **权限**：首次进入点"允许"摄像头与麦克风。
4. **模型**：默认已接入**火山方舟豆包大模型**（对话更聪明、药品视觉更准）。
   - ⚠️ 当前提供的 API Key 经测试**未通过鉴权**（服务端提示 key 不存在），请在方舟控制台核对后，于右上「设置」重新填入「API Key」及你开通的「文本/视觉模型 ID」（如 `doubao-seed-1-6-250615` 或推理接入点 `ep-xxxx`）。
   - 若豆包暂不可用，系统会**自动降级到免费模型**继续演示，**不会中断**——所以即便 Key 没配好也能正常演示。
5. **网络**：建议现场准备手机热点备用；降级到免费模型时若限速，说话可稍作停顿。
6. **音量**：建议外接音箱，数字人语音更有临场感。戴/不戴耳机均可，本系统支持"说话打断"。

> 演示小贴士：本系统的危机识别与药品错服判断内置了**本地规则兜底**，即使断网也能稳定复现关键剧情，不依赖模型联网。数字人 3D 模型与引擎均已**打包到本地 `assets/`**，现场不依赖外部 CDN。

## 四、三大演示场景与体验路径

演示建议顺序：**A 情绪陪伴 → B 心理危机干预 → C 药事管理**（由轻到重、层层递进）。
每个场景在顶部 Tab 一键切换。

### 场景 A：情绪陪伴 + 情绪识别（多模态）
**展示能力**：情绪识别模型（多模态）+ 情绪陪伴 AI 助手
**一句话看点**：数字人能"看脸色、懂情绪、会共情"。

体验路径：
1. 点「开始演示」，开启摄像头。数字人「乐颐」主动问候。
2. 演示者对着摄像头微笑，说：**"今天天气不错，我心情挺好的。"**
   - 右侧决策面板弹出【情绪识别模型】卡片：情绪标签 `愉悦/happy`、置信度、证据（文本正向词 + 表情识别结果）。
   - 乐颐用愉快语气回应，一起高兴。
3. 演示者做出难过表情，说：**"就是有点想家，有时候觉得孤单。"**
   - 决策面板弹出情绪标签 `孤独/lonely`、风险 `P2 关注`，证据含"表情：难过 + 文本负向词：孤单"。
   - 乐颐切换为关心神情，温柔安慰并引导聊家人/往事。

预期效果：领导看到"表情 + 语义"双通道融合，且每次判断都有证据可解释。

---

### 场景 B：心理危机干预（重头戏）
**展示能力**：情绪识别 → 心理危机干预 Agent → 服务调度推荐 Agent（三 Agent 联动）
**一句话看点**：识别到危机信号，秒级分级并触发"通知家属/护理员"的人工介入闭环。

体验路径：
1. 切到「危机干预」Tab。乐颐以陪伴姿态开场。
2. 先说一句一般低落：**"这几天晚上老是睡不着，心里闷得慌。"**
   - 决策面板：心理风险 `P2`，建议护理员今日关怀。乐颐温柔倾听。
3. 再说出**危机信号台词**：**"唉，活着没意思，我不想活了。"**
   - 决策面板连续弹出：
     - 【心理危机干预 Agent】：等级 `P0 紧急`（或 P1），证据="命中危机词库"，`need_human_review = true`。
     - 【服务调度推荐 Agent】：生成 `P0` 紧急上门任务，匹配最近护理员，SLA 2 分钟，通知家属+心理咨询师。
   - 乐颐**不说教**，用安全话术稳稳陪伴："我在这儿，别怕，我一直陪着您……我这就请工作人员过来陪您。"
   - 屏幕弹出**红色告警弹窗**，动态显示"值班护理员/家属/心理咨询师"逐一通知送达。
4. 点弹窗「已知悉，人工接管」结束该剧情。

预期效果：体现"AI 识别 + 分级 + 安全话术 + 人工介入闭环"的完整安全设计，突出"宁可误报不可漏报"和"高风险强制人工"的原则。

> 危机识别为**本地规则引擎兜底**，现场必现，不受网络影响。

---

### 场景 C：药事管理
**展示能力**：药事管理 Agent（+ 健康预警关联）
**一句话看点**：数字人能"看药、核药、拦错药"。

体验路径（两种方式，任选或都演示）：
1. 切到「药事管理」Tab。乐颐提示："今天上午该吃降压药氨氯地平一片，把药拿到镜头前我帮您核对。"
2. **方式一（视觉识别）**：手持药盒/药片对准摄像头，点底部「识别药品」。
   - 系统调用视觉模型识别药品名，与用药计划比对。
3. **方式二（模拟卡片，稳定兜底）**：点击底部药品托盘里的卡片。
   - 点「氨氯地平」→ 决策面板 `P3 常规`，核对一致，乐颐确认可服用，卡片变绿。
   - 点「二甲双胍」→ 决策面板 `P1 高风险`，识别不一致，`need_human_review=true`，乐颐**语音阻断**"先别吃"，弹出**错服预警**并通知护理员核实，卡片变红。

预期效果：展示"识别—匹配—阻断—人工确认"的用药安全闭环。

## 七、常见问题与兜底

- **没声音/不说话**：检查是否允许麦克风；或点键盘图标改用打字输入。
- **数字人不出声**：系统 TTS 依赖浏览器语音，Chrome/Edge 最佳；可在设置里选中文音色。
- **数字人显示为卡通头像（非 3D）**：说明 WebGL/3D 模型未加载成功，系统已自动回退到 SVG 卡通数字人兜底，功能不受影响；刷新或换 Chrome/Edge 通常即可恢复 3D。
- **豆包报错/很慢**：系统会自动切免费模型继续；如需用豆包，请在设置核对 API Key 与模型 ID。
- **摄像头打不开**：确认用 localhost/HTTPS 打开并授权；也可点「仅语音/文字模式」跳过摄像头。
- **视觉识别药品不准**：改用底部「模拟药品卡片」演示，效果稳定。
- **危机/错服剧情**：内置本地规则，断网也能稳定复现。

## 八、技术说明（备询）

- 前端单页应用，零后端依赖：`index.html` + `app.js`（业务逻辑）+ `avatar3d.js`（3D 数字人）+ `assets/`（3D 引擎与模型）。3D 资源全部本地化，可离线打开（模型调用需联网）。
- **3D 数字人**：Three.js 渲染写实人脸（含 52 个 ARKit 面部 blendshape）；口型由语音播放实时驱动（说话时 `onboundary` 脉冲 + 下颌开合动画），叠加眨眼、呼吸、头部微动与情绪表情；WebGL 不可用时自动回退 SVG 卡通数字人。
- 情绪识别：`face-api.js` 浏览器本地推理。
- 语音识别 / 合成：浏览器 Web Speech API。
- 对话大模型：OpenAI 兼容接口，默认接入**火山方舟豆包**（`ark.cn-beijing.volces.com/api/v3`），调用失败自动降级免费模型，保证演示不中断。
- 视觉药品识别：多模态模型（image_url 通道，默认豆包 vision），带免费视觉降级与本地卡片兜底。
- 危机识别：本地关键词规则引擎（对应 PRD 危机词库）+ 大模型语义，双保险。
- 决策面板严格对齐 PRD 统一输出 Schema：`risk_level / confidence / evidence / suggested_actions / need_human_review`。
